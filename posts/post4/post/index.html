本文主要是风辰大佬的《并行算法设计与性能优化》这本书的笔记

并行计算基础知识

基础概念

  • 多核、多路、众核
    同一个芯片多个核心为多核，多个多核芯片叫做多路，几百几千个核心集成在同一个硅片上叫众核。

  • 超线程（HT，Hyper-Threading）
    同一个物理核模拟成两个逻辑核，可并行执行两个线程，能够在单个时钟周期内进行线程间切换。

  • 指令级并行
    流水（pipeline）、乱序执行、指令多发射、分支预测（选择一条指令执行，一旦选择错误，就丢弃已经执行的结果）。

  • 向量化并行
    SIMD、SIMT

  • 线程级并行
    伪共享：多个线程读写的数据映射到同一条缓存线上时，如果一个线程更改了数据，那么其他线程对该数据的缓存就会失效，硬件就需要去更新缓存线。

  • 缓存
    A data：Intel Haswell cpu：一级缓存大小为32kb，延迟为3个时钟周期，二级缓存为256kb，延迟为11个时钟周期，一次内存访问的延时需要200时钟周期以上。

  • 缓存一致性
    伪共享产生的原因

  • 缓存不命中
    1、冷不命中，2、满不命中，3、冲突不命中（新读取的数据将会放在缓存中的某个地址a，a是由算法决定的，如果a的原始数据被访问，那么也会不命中，称为冲突不命中）

  • NUMA（非均匀内存访问） 核心访问离其近的内存延迟较小

性能度量

  • 度量标准：
    1、时间
    2、FLOPS，每秒执行的浮点运算的数量
    3、CPI，每次循环耗费的时钟周期数
    4、延迟，一个指令从发射到完全结束
    5、吞吐量，A，每时钟周期指令数量（IPC）B，每秒执行指令数量
    6、little定律：发挥硬件的某种指令的计算能力所需要的并行度等于指令的延迟乘以指令的吞吐量。

串行代码优化

  • 受限指针
    受限指针表示该指针不会和其他指针存在存储器别名
    看如下代码
int f(int *a, int *b) {
    *a += *b;
    *a += *b;
}

// 如果a和b指向不同地址，那么代码可以被优化为

int f(int *a, int *b) {
    int temp = *b;
    *a += 2 * temp;
}

// 如果a和b指向相同的地址，那么代码就变为

int f(int *a, int *b) {
    int temp = *b;
    temp += temp;   
    temp += temp;
    *a = temp；
}

// 也就是
int f(int *a, int *b) {
    int temp = *b;
    *a = 4 * temp;
}

 // 通过c99关键字restrict可以告诉编译器这是受限的指针，
 // 那么编译器就可以进行上文的优化，  
 // 否则，编译器无法优化，因为可能会产生错误的结果。
 // c++的关键字则为__restrict__

  • 索引顺序 C语言中数字是以行为主序存放的，而Fortran中则是以列为主序存放，遍历数组时需要注意这些点。

  • 减少内存读写 前文说过ddr3内存一次读写大概需要200多个时钟周期，因此应当优先使用数值而不是解引用。 如下面的代码优化

for(int i = 1; i < n; ++i) {
    a[i] += a[i - 1];
}

temp = a[0];
for(int i = 0; i < n; ++i) {
    temp += a[i];
    a[i] = temp;
}

  • 分支预测
    1、避免把判断放入循环中
    2、拆分循环
    3、合并多个条件 4、生成条件状态吗生成掩码来一处条件分支（判断变计算）

  • 优化乘除法和模余
    位运算最多只要一个周期，乘法需要三个周期，除法十几个，模余需要几十上百个

并行代码优化

  • MPI（消息传递接口）
    主要用于分布式存储的并行机。

  • OpenMp
    Open Multi-Processing，用于共享存储器的并行环境。

  • fork/pthread

  • CUDA

  • OpenCL

  • volatile
    不将其保存在缓存中，每次读的时候都直接去该变量的内存地址去读，每次写也是。可以避免寄存器或缓存对内存读写的优化。
